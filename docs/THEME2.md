# 解決したい課題 / 潜在的なニーズ

- 生産記録がすべて手書き：切断作業の記録は紙の台帳のみで、デジタルデータとして残っていない。後から検索・集計ができず、出荷時には種類がバラバラに書かれた紙を見ながら自分で重量を計算しており、時間もミスのリスクも大きい。
- 領収書・経費の処理が手作業：領収書はファイルに溜まっていくだけで、出納帳を見ながら手入力している。PDFやスキャン画像をExcelに転記する作業自体が二重入力になっており、入力ミスがあっても気づきにくい。
- 顧客への一斉案内手段が現状ない：新規開拓をしたいと考えおり、販促のスピード感を出すためにあらゆる連絡手段(電話・メール・LINEなど)で一斉配信、配信停止の管理の管理をしたい。

# 対象ユーザー（誰の、どんな状況か）

- 廃品回収・スクラップ加工を営む友人の会社で働く、現場作業員と事務スタッフです。現場作業員は切断作業をしながら紙に記録をつけ、出荷時には事務スタッフが紙を見ながら手計算をしています。ITに詳しいわけではないものの、LINEやWeChatは日常的に使っており、スマホやiPadの操作自体には抵抗がない、という人物像を想定しています。

# プロダクトのカテゴリー

- 中小企業向け業務管理システム(バックオフィス／現場業務デジタル化プラットフォーム)

# 提供する価値（ユーザーが得られる変化・体験）

- 紙に書いていた生産記録は、複数の品目をまとめて入力できるようになり、品目ごとの累計が記録するたびに自動で積み上がる(台帳をそのままデジタル化したイメージ)。出荷時の重量計算が要らなくなる。
- 領収書はOCRでテキスト化され、確定金額の管理状況（要確認／確定済み）がひと目で分かる。
- 連絡先は電話・メール・LINE・FAXといった複数の連絡手段をまとめて管理でき、配信停止もチャネルごとに設定できるので誤送信を防げる。
顧客への一斉配信は、SMS・メール・LINE・FAXを同じ画面で切り替えて送れる。

# 価値を実現するための最小限の機能（MVP）

フェーズ1(プロトタイプで確認中)

- 品目マスタ管理（使用中の品目は削除不可）
- 生産記録（複数品目のまとめ入力、品目別累計の自動表示）
- 経費データ管理（OCR金額・確定金額の管理、状態別の絞り込み）
- 連絡先管理(電話・メール・LINE・FAXをチャネルごとに登録、チャネル別の配信停止設定)
- 一斉配信(SMS・メール・LINE・FAXを画面上で切り替えて送信。電話・FAX・LINEは外部サービスとの契約が前提)

フェーズ2(今後検討)

業務メモ(写真添付のメモをOCRでテキスト化し、検索可能にする)
権限管理(経費データ・一斉配信はadmin権限のみアクセス可能にする)

# 既存の代替手段・競合との違い

- 紙台帳・Excel手入力との違い：記入した時点でデータ化され、二重入力・転記ミスがなくなる。
- 会計ソフト(freeeなど)との違い：仕訳重視の汎用ソフトではなく、現場の生産記録やスクラップ単位の集計など、この業務特有の業務フローにも対応可。
- 既存の配信サービスとの違い：チャネルごとに別々のツールを使うのではなく、自社の連絡先データと連動させた一斉配信を1つの画面で行える。

# 技術的にこだわりたい・工夫したい点（無理ない範囲で）

- 品目マスタを正規化して分離し、使用中の品目は削除できない制約をかける丁寧なDB設計。
- 連絡先は1件につき複数の連絡チャネルを持てる構造にし、チャネルが増えてもテーブル構造を変えずに拡張できるようにする。
- 領収書OCRの読み取り精度に配慮しつつ、読み取れなかった場合の手動修正フローも用意する。
- 現場で使うことを想定し、iPad・スマホでも見やすくレスポンシブ化対応。
- 経費データ・一斉配信は顧客の連絡先情報を扱うため、admin権限のみアクセスできるようアクセス権限管理に配慮する。
